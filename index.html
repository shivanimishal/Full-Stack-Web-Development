3. AP In-Flight Invoice Ageing — Alteryx Workflow

github.com/shivanimishal/alteryx-workflow-documentation

What This Workflow Does

Three internal AP systems each hold in-flight invoices but use completely different field names, status vocabularies, and ageing definitions. This Alteryx workflow standardises all three streams into a single canonical schema, identifies overdue invoices, and assigns a P1–P4 priority flag with a responsible team to every invoice.

  1. Reads invoice data from three source systems (Alpha ERP, Beta P2P Platform, Gamma AP Tool)
  2. Standardises every stream to a canonical schema using Select tools
  3. Unions the three streams into one combined dataset
  4. Filters to in-flight invoices only
  5. Calculates days past due with per-system ageing thresholds
  6. Assigns responsible team and P1–P4 priority flag

Source Systems & Ageing Thresholds

SystemTypeAgeing ThresholdRationale
AlphaLegacy ERP> 30 days = Aged - OverdueStandard threshold
BetaProcure-to-Pay Platform> 21 days = Aged - OverdueStrictest — most automated routing
GammaAP Automation Tool> 45 days = Aged - OverdueMost lenient — multi-tier sign-off

5. Report Validation Toolkit

github.com/shivanimishal/report-validation-toolkit

Business Problem

When migrating from a legacy reporting system to a new automated pipeline, how do you systematically validate that outputs match? This tool automates the comparison process.

What It Does

  • Row count comparison between legacy and new reports
  • Column schema validation (missing or added columns flagged)
  • Null value change detection per column
  • Numeric variance detection — flags columns where values differ by more than a configurable threshold (default: 5%)

7. Athena — Women's Safety Wearable Device

github.com/Aditi-Chowdhuri/Athena  |  Built at Hack36  |  Demo Video  |  Presentation

Team: Shivani Mishal · Aditi Chowdhuri · Trisha Gupta · Sakshi Parikh

Problem Statement

India reports nearly 90 rape cases per day, with a conviction rate of just 32%, largely due to lack of tangible evidence. Existing safety apps require the device to be unlocked and opened — too slow in an emergency. Athena is a wearable neckpiece that activates with 3 taps, simultaneously triggering evidence capture, SOS messaging, and a police tracking dashboard.

What Athena Does

  • Evidence capture: Hidden OV7670 camera module takes images at fixed intervals; saved to SD card for use in court proceedings
  • SOS messaging: Flutter app sends emergency SMS and current GPS location to saved contacts and helplines
  • Police dashboard: Web interface using Google Maps API marks all active distress signals to help police coordinate response
  • Self-defence: Taser module generating up to 400kV for attacker immobilisation; Rs 500–600 total BOM cost

8. Justice — Emergency Safety App & Hardware

github.com/Aditi-Chowdhuri/Justice  |  DSC Hackathon  |  App Demo  |  Hardware Demo

What It Does

A two-app system (user-facing and police-facing) backed by Bluetooth-enabled hardware for emergency response. Hitting EMERGENCY in the app simultaneously:

  • Sends SMS to saved personal contacts and the emergency helpline number
  • Calls the emergency helpline automatically
  • Starts video recording on the phone
  • Broadcasts the user's live GPS location to the Police 2.0 dashboard, shown as a map marker

Components

  • Justice App — user-facing: contact management, one-tap emergency trigger
  • Police 2.0 App — police-facing: live map showing users in distress
  • Bluetooth Hardware — PCB-level design with Bluetooth module for device pairing

Skills: Android development · Bluetooth integration · PCB design · Maps API · Emergency systems UX

This project and Athena are complementary: Justice focuses on the software/app layer and police coordination; Athena adds wearable hardware for hands-free activation and evidence capture.
